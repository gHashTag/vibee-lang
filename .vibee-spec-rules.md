## CRITICAL: YAML Syntax!

**ALL .vibee spec files MUST use YAML syntax!**

### ‚úÖ CORRECT (.vibee with YAML):

```yaml
# specs/example.vibee
name: example
version: "1.0.0"
language: gleam
module: example

types:
  - name: User
    kind: record
    fields:
      - name: name
        type: String
      - name: age
        type: Int

functions:
  - name: create_user
    visibility: pub
    params:
      - name: name
        type: String
      - name: age
        type: Int
    return_type: User
    body: |
      User(name: name, age: age)

tests:
  - name: create_user_works
    body: |
      let user = create_user("Alice", 30)
      user.name == "Alice"

imports:
  - gleam/string
  - gleam/list
```

### ‚ùå WRONG (Custom syntax):

```vibee
spec example {
  version: "1.0.0"
  types { ... }
  functions { ... }
}
```

## Why YAML?

1. **Standard format** - Everyone knows YAML
2. **Easy parsing** - Libraries available
3. **Readable** - Clear structure
4. **Tooling** - Editors support YAML
5. **Validation** - Schema validation possible

## File Extension

- **File extension**: `.vibee` ‚úÖ
- **Syntax**: YAML ‚úÖ
- **Compiler**: `vibeec` ‚úÖ
- **Command**: `vibee gen` ‚úÖ

## Remember

- `.vibee` = vibee language files (YAML syntax)
- `vibeec` = vibee compiler (the tool)
- `vibee gen` = command to generate code
