## Описание

Настройка и использование Sentry для мониторинга ошибок и производительности проекта Vibee. Трейсинг, алерты, анализ ошибок.

## Основные возможности

- **Error Tracking**: Отслеживание и анализ ошибок в production
- **Performance Monitoring**: Мониторинг производительности приложения
- **Tracing**: Распределенный трейсинг запросов
- **Alerts**: Настройка алертов на критические ошибки
- **Releases**: Отслеживание деплоев и их влияние на ошибки

## Примеры использования

- Настройка Sentry для отслеживания ошибок
- Анализ ошибок в production
- Настройка алертов на критические проблемы
- Интеграция с деплой процессом для отслеживания releases

## Связанные скиллы

- `diagnostics-performance` - анализ производительности
- `langfuse-observability` - общая observability
- `cicd-pipeline` - интеграция с деплоем

## Best Practices

- Настраивай Sentry для всех окружений
- Группируй похожие ошибки для анализа
- Используй source maps для отладки
- Настраивай алерты на новые типы ошибок
- Отслеживай performance budgets
- Интегрируй с деплоями для анализа регрессий
