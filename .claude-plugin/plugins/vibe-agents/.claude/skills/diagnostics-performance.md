## Описание

Диагностика проблем производительности и оптимизация системы Vibee. Профилирование, поиск узких мест, анализ метрик.

## Основные возможности

- **Профилирование**: Анализ производительности кода и выявление узких мест
- **Метрики**: Сбор и анализ метрик производительности
- **Health Checks**: Проверка здоровья системы и компонентов
- **Оптимизация**: Улучшение производительности на основе анализа
- **Мониторинг**: Непрерывное отслеживание производительности

## Примеры использования

- Анализ медленных запросов к LLM API
- Оптимизация работы плагинов ElizaOS
- Поиск узких мест в обработке сообщений Telegram
- Настройка health checks для критических компонентов

## Связанные скиллы

- `monitoring-sentry` - мониторинг ошибок и производительности
- `langfuse-observability` - метрики LLM операций
- `code-review` - анализ качества кода

## Best Practices

- Регулярно профилируй критичные части системы
- Устанавливай baseline метрики для сравнения
- Используй автоматические алерты на деградацию производительности
- Документируй найденные проблемы и решения
